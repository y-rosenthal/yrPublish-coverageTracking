I would like the coverage-data to be more readable and maintainable by humans. Therefore, don't use JSON for the coverage-data file.
Also use a different file coverage-data file for each class section. These should be CSV files that contain lines such as:

URL,html-id,covered

or

URL,html-id,not-covered

If there is no entry for a particular URL,html-id in one of these csv files then the default logic is that it was not covered.
However, if a line in the csv appears as:

URL,html-id

or

URL,html-id,

then the default should be tha it WAS covered.
This allows for these coverage-data csv files to be easier to read and edit by a professor by listing all content sections in the book.
Alternatively, for bigger books it might be nicer to have a smaller file by not-listing some content sections that were not covered yet.

If a new section is added to the text without updating the coverage the coverage will symantically indicate that it wasn't covered yet.

Please, continue to use the JSON for the coverage-config.json file.

## tools

Also I would like some tools for professors and content editors.

1. A bash script named yrAddSectionsIds.sh, that takes a .qmd file as input and outputs a new version of the .qmd that adds html-ids to any section that didn't already have one. Make sure to skip quarto code blocks so that # signs in the code block aren't mistaken for Quarto headings.

2. A bash script named yrUpdateCoverage.sh that takes one or more .qmd files as input as well as a coverage-data csv file (as described) for a section. The script should be assumed to be run from inside a Quarto project that uses this system for the resolution of URLs. The script should output a new .csv file that adds any missing html-ids for the .qmd files.

3. Design a way for a professor to use the website's UI to create and edit these .csv files. There should be a way to
a) select all html-ids on the page as having been covered
b) de-select all html-ids on the page
c) individually select or de-select specific html-ids on the page
d) write the selection data out to a csv file. If that is not possible then give the professor a way to display the contents of the coverage-data csv to a popup or similar so that the professor can copy/paste the info
into a csv and save/update it while offline.
